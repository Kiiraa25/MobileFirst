Framework, qu'est ce que c'est ?

= Cadre de travail


un framework agit comme une librairie ou une boîte à outils, fournissant des composants et des éléments pré-stylisés prêts à être utilisés.
Cela simplifie le travail des développeurs en réduisant le temps nécessaire pour créer certains éléments.
Cependant, l'utilisation d'un framework implique également de suivre un ensemble de règles et de contraintes, car il offre un cadre de travail défini.
Cela peut apporter une efficacité notable tout en imposant une structure cohérente au développement web.

Comment ça fonctionne ?


Un framework c'est en réalité un ensemble de codes réutilisables.

Il est donc toujours associé à un langage de programmation puisque les éléments mis à disposition ont été codés dans le langage auquel il est associé.

S'approprier l'utilisation d'un framework peu prendre du temps. il est aussi nécessaire de maitriser un minimum le langage auquel il est associé avant de s'y aventurer, mais il n'est pas non plus nécessaire d'être un expert en css pour pouvoir utiliser bootstrap par exemple.

des connaissances de base suffisent.



Pour pouvoir utiliser bootstrap par exemple :

On va aller sur le site officiel de Bootstrap qui va nous fournir un code à copier, et coller dans la balise <head> de notre HTML.

Ce code nous permettra d'avoir accès, et utiliser les élements mis à disposition par le framework.

Il nous suffira ensuite de copier dans notre HTML le code correspondant à l'élement que nous souhaitons utiliser.

Avantages

de l'utilisation d'un Framework

- Gain de temps
On bénéficie d'un gain de temps considérable en fonction des éléments que l'on souhaite utiliser. Il n'est effectivement plus nécessaire de coder les éléments mis à disposition par le Framework.

- Un code propre et organisé :
Un framework va imposer une structure et de bonnes pratiques de développement.

- Un design cohérent :
les frameworks favorisent la cohérence du design en fournissant des directive de style uniforme.
le but étant de permettre une expérience utilisateurs optimale.

- Responsive design : les frameworks intègrent souvent des systèmes de grille afin de permettre aux pages web de s'adapter facilement aux différentes tailles d'écran.
Les éléments importés d'un framework sont donc souvent déja responsive.

- compatibilité inter-navigateurs
les framework ont été pensé de sorte a ce qu'un unique code ai exactement le même rendu peu importe le navigateur sur lequel on consulte.

- Communauté active :
Les frameworks disposent qu'une communauté active, qui peut fournir de la documentation et des ressources supplémentaires.

Inconvénients

de l'utilisation d'un Framework

- personnalisation limitée :
Bien que les frameworks CSS soient souvent personnalisables, il peut y avoir des limitations quant à la modification de certains aspects.
Il devient necessaire de bien maitriser le langage associé et maitriser le fonctionnement des frameworks si on souhaite faire des modifications plus complexes.

- Tailles de fichiers :
les fichiers générés a partir de frameworks sont plus volumineux et peuvent augmenter le temps de chargement des pages.

- Flexibilité limitée :
les frameworks peuvent imposer des cadres rigides, et certains projets ne peuvent pas bénéficier pleinement de l'utilisation d'un framework (notamment les projets necesitant un haut niveau de personnalisation).
Il peut parfois être plus judicieux de travailler sans framework.
